# AI Game Lab

초등학생 자녀와 함께 Codex 또는 Claude Code로 아이디어를 실제 게임으로 바꾸기 위한 로컬 프로젝트입니다.

현재 첫 게임은 Minecraft/Roblox 취향을 반영한 3D 1인칭 야생 생존 게임입니다.

## 지금 들어 있는 것

- Vite + TypeScript + Three.js 기반의 브라우저 3D 게임 환경
- React 기반 메뉴/설정 화면을 붙일 수 있는 준비
- 바로 실행 가능한 1인칭 야생 생존 게임
- 아이디어를 정리하는 문서 템플릿
- AI에게 변경을 요청할 때 쓸 수 있는 프롬프트 예시

## 실행 방법

PowerShell에서 아래 명령을 실행합니다.

```powershell
cd C:\ai-game-lab
.\start-dev.ps1
```

브라우저에서 아래 주소를 엽니다.

```text
http://127.0.0.1:5173
```

PC를 재시작한 뒤에는 보통 아래처럼 짧게 실행할 수 있습니다.

```powershell
cd C:\ai-game-lab
npm run dev
```

설치 직후 `Access is denied`가 보이면 새 PowerShell을 열거나 `.\start-dev.ps1`을 사용하면 됩니다.

## 현재 게임 조작

- 화면 클릭: 1인칭 시점 고정
- 마우스 이동: 시점 회전
- `WASD`: 이동
- `Shift+W`: 달리기
- `Shift`: 웅크리기
- `C`: 엎드리기
- `Space`: 점프
- `1-8`: 핫바 선택
- `E`: 보고 있는 대상과 상호작용
- `I`: 인벤토리와 2x2 미니 제작대
- `B`: 튜토리얼 책
- `P`: 보유 중인 제작대/제련대 설치
- `Ctrl+S`: 저장
- `Ctrl+L`: 불러오기
- `N`: 타이틀 화면으로 돌아가기 (새로시작)
- `Esc`: 창 닫기 또는 시점 고정 해제

화면 왼쪽 위의 `저장`, `불러오기` 버튼으로도 같은 기능을 사용할 수 있습니다. `새로시작` 버튼은 확인 후 타이틀 화면으로 돌아가며, 타이틀에서 직업을 고르고 새 게임을 시작합니다.

## 현재 구현된 생존 규칙

- 처음 핫바는 4칸이고 튜토리얼 책 1권이 들어 있습니다.
- 가방을 만들면 핫바가 8칸, 인벤토리가 40칸으로 늘어납니다.
- 작은 나무는 맨손으로 캘 수 있고 나무 1개를 줍니다.
- 큰 나무는 도끼가 필요하고 나무 5개를 줍니다.
- 작은 나무가 더 자주 나오며 큰 나무와 크기/색/형태로 구별됩니다.
- 구조물, 나무, 동물, 주민, 기사와 충돌합니다. 낮은 구조물은 점프로 넘을 수 있습니다.
- 흙 지역은 손이나 삽으로 팔 수 있고, 파다 보면 돌층이 나와 더 이상 팔 수 없습니다.
- 올라갈 수 있는 산 지형이 중간중간 배치됩니다.
- 100걸음마다 50% 확률로 상자가 생깁니다.
- 500걸음마다 20% 확률로 동굴이 생깁니다.
- 상자는 망치 50%, 제련대 2% 확률을 가집니다.
- 제작대, 확장 제작대, 제련대, 특수 제련대, 도구, 무기, 갑옷, 가방 제작 흐름이 들어 있습니다.
- 동굴은 야생 필드와 다른 어두운 배경, 둥근 돌벽, 입구/끝 출구를 가집니다.
- 동굴에는 돌, 석탄, 드문 광부, 아주 낮은 확률의 광산 상자가 있습니다.
- 흑요석은 다이아몬드 곡괭이로만 캘 수 있고 특수 제련대에서만 제련됩니다.
- 마을에는 주민, 주민의 집, 우물, 식량창고, 체력 10/방어 5의 마을기사가 있습니다.
- 한 마을 식량창고에서는 고기를 최대 10개까지 가져올 수 있고, 가져오면 기사들이 공격합니다.
- 나무 캐기, 흙 파기, 공격, 설치 같은 액션에는 1인칭 손 모션이 나옵니다.
- 보스는 챕터 순서대로 도전합니다: 용 → 파이어 드래곤 → 레드 드래곤 → 레이저 드래곤 → 다크 드래곤 → 불멸의 존재. 이전 보스를 처치해야 다음 보스의 봉인이 풀리고, 화면 목표에 현재 챕터와 권장 레벨이 표시됩니다.
- 야생 몬스터는 31종입니다. 거미/늑대/사자 계열에 더해 멧돼지(돌진), 뱀(런지), 박쥐(급강하), 전갈(꼬리 찍기), 곰(일어서서 내려찍기), 좀비(양팔 휘두르기), 귀신(반투명 돌진), 새끼용(날아 덮치기) 계열이 레벨 8~110 구간을 채웁니다.
- 근접 몬스터의 공격은 2단계로 또렷하게 보입니다: 웅크리고 뒤로 빠지며 부르르 떠는 예열 후, 플레이어 쪽으로 크게 도약하며 덮칩니다. 떨림이 보이면 피할 준비를 하세요.
- 용용 평원(Lv 10~25)은 새끼용 4종(풀빛·바람·바위·황금)과 다양한 몬스터가 우글대는 사냥 전용 맵입니다. 나무와 동굴이 드물고, 처치 경험치를 3배로 줍니다.
- 공동묘지 원정 맵(Lv 70~85)에는 좀비·구울·묘지귀신·무덤박쥐가 출몰하고, 낮에도 하늘이 어둑한 음산한 분위기입니다.
- 공동묘지의 무덤 약 50기 중 1기에는 초록 좀비 손이 솟아 있습니다. 그 위를 지나가면 좁은 지하 무덤으로 끌려가 좀비 1마리와 싸워야 하며, 이기면 출구가 열리고 지면 일반 사망 규칙을 따릅니다.
- 저장/불러오기는 브라우저 localStorage를 사용합니다. 같은 PC와 같은 브라우저에서 이어 할 수 있습니다.

## 추천 진행 방식

1. `docs/game-idea-canvas.md`를 아이와 같이 채웁니다.
2. Codex에게 "이 아이디어를 현재 Three.js 1인칭 야생 게임에 반영해줘"라고 요청합니다.
3. `npm run dev` 화면에서 바로 놀아봅니다.
4. 한 번에 한 가지씩 바꿉니다. 예: 블록 설치, 동물 이동, 동굴 확장, 저장 기능.
5. 재미있어진 순간을 저장하려면 Git 커밋을 만듭니다.

## 아이와 함께할 때 좋은 약속

- 개인 정보, 학교 이름, 친구 이름은 게임 안에 넣지 않습니다.
- AI 답변은 정답이 아니라 초안으로 봅니다.
- 아이가 만든 규칙을 먼저 존중하고, 어른은 구현을 도와주는 역할을 합니다.
- 실패한 아이디어도 기록합니다. 게임 만들기에서는 실패가 좋은 재료가 됩니다.
