# 파티 시스템 설계 (2인 이상 협동 플레이)

> 상태: 1차(닉네임)·2차(접속/로비)·3차(프레즌스)·4차(소셜)·5차(월드 공유)·5.1(다듬기)·6차(마을 경비)·**7차(동굴·보물상자 공유) 완료**.
> 관련 코드: `src/game/nickname.ts` · `src/game/party.ts` · `src/ui/partyPanel.ts` · `.github/workflows/deploy.yml`

## 1. 목표
연우와 친구(2~4명)가 **각자 자기 컴퓨터에서** 같은 월드에 접속해 파티를 맺고 함께 클리어한다.

## 2. 접속 방식 비교 — 결정 기록

| 방식 | 장점 | 단점 | 판정 |
|---|---|---|---|
| **URL 공유 (웹 배포)** | 무설치(브라우저만) · 무료 · push 즉시 업데이트 · 초등학생 친구도 링크 클릭이면 끝 | 인터넷 필요 | ✅ **채택** |
| 앱스토어/스토어 배포 | 아이콘 설치감 | 개발자 계정 비용(연 $99+)·심사 수일·업데이트마다 재심사·패키징(Electron 등) 추가 작업, **그래도 네트워크 인프라는 별도로 필요** | ❌ 반려 |
| 로컬 LAN 직결 | 서버 불필요 | 같은 와이파이에서만 — "친구 집에서 접속" 불가 | ❌ 반려 |

**배포 = GitHub Pages.** 저장소가 이미 GitHub 에 있어 Actions 로 자동 배포되며, 주소는
`https://seohyunbum.github.io/YUNU_GAME/` 하나로 고정된다. 친구에게 이 링크만 보내면 된다.

## 3. 멀티플레이 연결 구조 — 결정 기록

| 구조 | 장점 | 단점 | 판정 |
|---|---|---|---|
| **WebRTC P2P (호스트-게스트)** | 서버 운영비 0 · 지연 최소 · 2~4인에 최적 | 시그널링(악수) 채널 필요 · 일부 공유기에서 직결 실패(~10%) | ✅ **채택** |
| 중계(WebSocket) 서버 | 가장 확실한 연결 | 서버 호스팅·운영 부담(계정, 배포, 비용/슬립), 모든 트래픽이 서버 경유 | 보류 — P2P 실패율이 문제 되면 폴백으로 |
| 시그널링 수동 복붙 | 완전 무서버 | SDP 블롭 복붙 — 아이들에게 비현실적 UX | ❌ 반려 |

**시그널링 = PeerJS 퍼블릭 브로커.** 무료 공용 서버가 악수만 중개하고 게임 데이터는 P2P 로 흐른다.
계정·배포·운영이 전혀 필요 없고, 코드상 `party.ts` 한 곳에 격리되어 있어 나중에
자체 PeerServer 나 Cloudflare 로 갈아타도 UI/게임 코드는 그대로다.

### 접속 UX (아이 기준)
1. 호스트: 타이틀 → **파티** → **방 만들기** → 6자리 초대 코드 표시 (예: `K7P2QX`)
2. 게스트: 같은 URL 접속 → **파티** → **코드로 참가** → 코드 입력 → 연결
3. 로비에 파티원 닉네임 목록 표시. (게임 내 합류는 3차)

## 4. 역할 모델: 호스트 권위 (host-authoritative)
- **호스트가 월드의 주인**: 몬스터·드랍·시간·보스 상태는 호스트 시뮬레이션이 정본.
- 게스트는 입력/위치를 보내고, 호스트가 검증·반영해 전파한다. (저장도 호스트 세이브에만)
- 2~4인 P2P 에서 구현 난도와 치팅 방지의 균형이 가장 좋은 모델.

## 5. 프로토콜 v1 (DataChannel·JSON)
| type | 방향 | 내용 |
|---|---|---|
| `hello` | 게스트→호스트 | `{ nickname, protocol }` 참가 요청 |
| `welcome` | 호스트→게스트 | `{ members[], hostNickname }` 수락 + 로스터 |
| `reject` | 호스트→게스트 | `{ reason }` (정원 초과·프로토콜 불일치·닉네임 중복) |
| `roster` | 호스트→전원 | 멤버 입퇴장 시 전체 목록 갱신 |
| `ping`/`pong` | 양방향 | RTT 측정·생존 확인 |
| (3차+) `state`/`input` | — | 위치·행동 동기화 |

- 정원: 호스트 포함 4명. 프로토콜 버전 불일치 시 거절(게임 버전 안내).
- 초대 코드: 혼동 글자(0/O/1/I) 제외 32자 알파벳 6자리 → PeerJS id `yunu-game-<코드>`.

## 6. 단계 로드맵
- **2차 (이번)**: GitHub Pages 자동 배포 + 파티 로비(방 만들기/코드 참가/로스터/핑). 게임플레이 동기화 없음.
- **3차 ✅**: 프레즌스 동기화 — 위치·회전 8Hz 송신(호스트 중계 스타 토폴로지), 같은 맵 파티원 아바타+머리 위 닉네임 렌더(보간), 지역 지도에 파티원 마커. 각자 월드는 독립 생성이라 지형 소품은 다를 수 있음(4차에서 호스트 월드 공유로 해소).
- **4차 ✅**: 소셜 계층(§9) — 유저 검색→친구 요청→팝업 승인→파티 요청→합류(타이틀=동시 시작 / 인게임=소환). 인게임 파티 패널 단축키 **O**.
- **5차 ✅**: 호스트 권위 월드 공유(§10) — 몬스터 동기화(8Hz 풀스냅샷)·전투 판정(호스트)·경험치 분배(같은 맵 전원 100%)·필드보스 공유.
- **6차 후보**: 용/잼미니/챕터보스 동기화, 드랍 오브젝트·상자 공유, 게스트 세이브 merge 정책, 귓속말.
- **폴백 트랙**: P2P 연결 실패 보고가 잦으면 무료 TURN(metered.ca) 추가 → 그래도 부족하면 중계 서버 전환.

## 7. 리스크
- **공용 PeerJS 브로커 다운**: 악수만 막힘(진행 중인 게임은 무관). 폴백: 자체 PeerServer 1줄 배포.
- **학교/공공 와이파이 방화벽**: WebRTC 차단 가능 — "연결 안 되면 휴대폰 핫스팟" 안내 문구로 1차 대응.
- **호스트 이탈 = 세션 종료**: 의도된 설계(호스트 권위). 로비에 명시.

## 8. 운영 체크리스트 (1회성, 저장소 주인)
- GitHub 저장소 Settings → Pages → Source 를 **GitHub Actions** 로 설정해야 첫 배포가 활성화된다.

## 9. 소셜 계층 (4차) — 친구 기반 초대

> 초대 코드 직접 입력 방식에서 **친구 시스템** 으로 개편 (코드 방식은 디렉터리 미설정 시 폴백으로 유지).

흐름: 파티 → 유저 검색(like) → **친구추가 요청**(상대가 접속 중일 때만) → 상대 화면 팝업(타이틀/인게임 무관) 승인 →
서로 친구 → 친구 목록에서 **파티 요청** → 상대 팝업 수락 → 요청자가 타이틀이면 둘이 같이 시작,
인게임이면 수락자가 요청자 곁으로 **소환**. (귓속말 버튼 자리는 친구 행에 예약돼 있음 — 추후 구현)

### 디렉터리 백엔드 (교체 가능 인터페이스 `directory.ts`)
| 백엔드 | 용도 | 범위 |
|---|---|---|
| `BroadcastDirectory` | 개발/테스트 + 같은 PC 두 탭 | 같은 브라우저 안에서만 |
| `FirebaseDirectory` | **실서비스** (서로 다른 집/PC) | 인터넷 전체 |

파티 "게임 데이터"는 여전히 PeerJS P2P 로 흐른다. 디렉터리는 명부·온라인 여부·요청 전달만 담당.

### Firebase 설정 (1회, 약 5분 — 저장소 주인)
1. console.firebase.google.com → 프로젝트 추가 (애널리틱스 끔)
2. 빌드 → **Realtime Database** → 만들기 (asia-southeast1, 잠금 모드로 시작)
3. 규칙 탭에 아래를 붙여넣고 게시:
```json
{
  "rules": {
    "users":  { ".read": true, "$nick": { ".write": true } },
    "friends":{ ".read": true, "$nick": { ".write": true } },
    "inbox":  { "$nick": { ".read": true, ".write": true } }
  }
}
```
   (아이 프로젝트 수준의 느슨한 규칙 — 인증 없이 닉네임 신뢰. 문제가 생기면 익명 인증 + 검증 규칙으로 강화)
4. 프로젝트 설정 → 웹 앱(</>) 추가 → `firebaseConfig` 값을 `src/onlineConfig.ts` 의 `FIREBASE_CONFIG` 에 붙여넣고 push
   (이 키는 공개 가능한 클라이언트 식별자다)

설정 전: 배포본 파티 패널은 초대 코드 방식만 노출. 설정 후: 친구 시스템이 자동 활성화.
(디렉터리 연결이 실패하면 친구 기능을 조용히 멈추지 않고 초대 코드 폴백으로 전환 + 상태 메시지 안내.)

### 엣지 케이스 처리 (멀티에이전트 검수 후 보강)
- **초대 수락 = welcome-확인 후 스왑**: 새 파티의 welcome 을 받은 뒤에야 기존 파티를 떠난다. 합류 실패 시 기존 파티 유지.
- **교차 초대**(둘이 동시에 파티 요청): 닉네임 사전순 작은 쪽 방을 살리고 큰 쪽이 자동 합류 — 죽은 방 교착 방지. 요청 의도는 클릭 즉시 기록해 레이스 창을 닫음.
- **호스트가 동굴/집 내부**: 게스트는 침묵 대기 대신 안내 메시지 + (타이틀이면) 자기 게임 선시작, 호스트가 야외로 나오면 곁으로 소환.
- **소환 흐름은 게스트 세션 전용**: 호스트가 게스트 좌표로 끌려가는 역소환 차단. 소환 시 수락자의 동굴/집 모드는 자동 해제.
- **파티 종료 시 잔존 상태 제거**: 소환 대기·보낸 요청 목록이 다음 파티로 새지 않는다 (`resetPartyFlow`).
- 게임이 자동 시작되면 파티 패널은 자동으로 닫힌다. 인게임에서는 **O** 키로 다시 연다.

## 10. 호스트 권위 월드 공유 (5차)

> 월드 난수(산·스폰·배회)가 비시드 전역 난수라 시드 동기화(lockstep)는 불가 — **호스트가 진실, 게스트는 뷰** 모델을 채택. 구현 = `src/game/partyWorldSync.ts` (프로토콜 **v2**).

| 메시지 | 방향 | 내용 |
|---|---|---|
| `mobs` | H→전원 | 현재 맵 야생 몬스터(필드보스 포함) **풀스냅샷 8Hz** — 게스트가 diff 로 스폰/이동/제거 전부 처리 (합류 스냅샷·리스폰·맵 전환이 공짜로 해결). 공격 모션(`atk/afx/afz`)도 실어 게스트가 같은 전조를 본다. 적체된 게스트는 스킵(백프레셔) |
| `attackRequest` | G→H | 게스트 공격(근접/원거리/스킬/도트/펫) — 게스트는 낙관적 표시 후 호스트 판정 대기. 호스트가 실내/타이틀이면 판정 보류 |
| `partyKill` | H→전원 | 처치 공유 — 같은 맵 전원 **100% XP**, 전리품은 처치자만 로컬 롤(드랍 오브젝트 동기화 회피), 필드보스 토벌 기록도 **같은 맵에서만** 공유. 동기화 범위(야생 몬스터)와 일치 — 용/잼미니 처치는 공유 안 함(이중 취득 방지) |
| `mobHit` | H→게스트 | 호스트 몬스터가 게스트를 타격(같은 맵에서 가장 가까운 파티원을 노린다). mapId 검증 + 사망 직후 3초 grace(연쇄 사망 방지) + 숨긴 탭 수신분 폐기 |

**설계 요점**
- 게스트의 동기화 몬스터는 진짜 `WorldObject` 로 스폰(조준·타격 가능, `partyTransient` 마킹으로 세이브/worldState 영속 금지). y 좌표는 보내지 않고 게스트 지형으로 재접지 — 산 난수 발산 회피.
- **게이트는 스냅샷 기반**: 게스트라도 "호스트와 같은 맵 + 스냅샷 신선(2초)"일 때만 로컬 시뮬(predatorAi·야생 스폰·야생 리스폰·필드보스 ensure)을 멈춘다. 호스트가 다른 맵/실내/타이틀/백그라운드면 게이트가 풀려 **자기 월드 로컬 사냥이 살아난다**. 경비·잼미니 리스폰은 파티 중에도 정상(호스트 비동기화 타입).
- 합류 시 로컬 몬스터는 **복원 정보를 자체 보관**(clearWorld 와이프에 면역) 후 silent 제거 → 동기화가 풀리면 프레임당 4마리씩 스태거 복원(히치 방지). 새 게임/로드(clearWorld)는 스냅샷의 liveness 검사가 감지해 자동 재동기화.
- 게스트 공격 가로채기는 `ProjectileDamageContext.partyAttackIntercept` 옵셔널 필드(combat.ts 는 partyWorldSync 를 모름 — 순환 의존 차단), DOT 는 classSkills, 소환사 펫은 summonerPet 에서 직접 우회. 데미지 공식은 본 전투와 동일(필드보스 armor 포함).
- 낙관적 hp 는 타격 후 600ms 동안 스냅샷의 상향 되돌림을 보류(하향 정정은 즉시) — 연타 시 체력 표시 출렁임 방지.
- 프레즌스가 6초 끊긴 게스트(백그라운드 탭)는 호스트 명단에서 제외 — 유령 좌표 영구 타격 방지. 패널(인벤토리) 연 게스트는 `panelOpen` 플래그로 로컬과 같은 타격 보류 보호.
- main.ts 배선은 전부 in-place 확장 — 라쳇 9489줄/459메서드 순증 0.

**알려진 한계 (7차 후보)**
- 용·잼미니·챕터보스·동물·드랍 오브젝트·상자는 각자 로컬 (게스트도 자기 것과 로컬 전투 가능, XP 공유 없음). → 같이 사냥 가능한 대상은 **야생 몬스터 + 필드 보스 + 마을 경비**(6차).
- 파티 중 게스트가 저장하면 그 맵의 원래 몬스터 배치는 세이브에서 빠진다(복원 스태시는 메모리) — 야간/지역 스폰이 자연 보충.
- 호스트 탭이 백그라운드면 rAF 정지로 스냅샷이 멈춘다 → 2초 후 게스트는 로컬 사냥으로 자동 전환, 호스트 복귀 시 재동기화.
- 호스트가 게스트 공격력을 신뢰(치팅 방지 없음) — 아이 협동 게임 트레이드오프.
- 파티 중 게스트 소환사 펫은 처치 XP 를 파티 분배로 받아 펫 레벨 분배(petXpShare)를 우회 — 펫 성장 밸런스 공백.

## 11. 실플레이 다듬기 (5.1)

2인 실플레이 피드백 6종 수정:
- **몬스터 분리 현상**: `applyMobsSnapshot` 단발 정리 가드가 첫 정리 이후 끼어든 로컬 몬스터(seedOverworld 잔여)를 못 치워, 게스트가 호스트 몬스터 + 자기 몬스터를 겹쳐 보던 문제 → **매 스냅샷마다 비동기화 야생 몬스터 제거**(복원 스태시는 1회만). 이제 게스트·호스트가 동일 집합을 본다.
- **친구 표시**: 머리 위 닉네임 표찰 절반 크기 + **HP바**(좌측 고정, scale/color 만 갱신해 핫패스 할당 0). `PresenceData.health/maxHealth` 8Hz 동기.
- **순간이동 개선**: 원격 아바타 보간을 속도 상한(11u/s)으로 제한 — 패킷 지연·유실로 멀리 온 갱신도 대시하지 않고 부드럽게 따라온다. 진짜 텔레포트(>24u)만 즉시 스냅. lerp 12/s.
- **힐러 파티 힐**: T(치유의 비)·R(천상치유) 둘 다 사정거리(12) 내 파티원에게 `partyHeal` 송신 → 수신자가 자기 HP 에 적용. 풀피여도 근처에 도울 친구가 있으면 발동(기존 풀피 차단 가드 제거).
- **친구 공격 표시**: 근접/원거리/스킬 시 `playerAttack` 브로드캐스트 → 친구 화면에 아바타 공격 모션(앞으로 숙임) + 원거리·스킬은 시각 전용 투사체 재생.
- **플레이어 충돌**: `resolveCollisions` 가 같은 맵 파티원과도 밀어내기(반경 0.42×2) — 완전히 겹치지 않는다.

전송은 호스트 중계(게스트→호스트→다른 게스트). 2인은 직결. main.ts 배선은 in-place 로 라쳇 9489/459 유지.

## 12. 마을 경비 동기화 (6차)

> 5차 호스트 권위 모델을 **마을 경비 4종(villageKnight/villageArcher/villageMage/villageGolem)** 으로 확장 — 둘이 같이 마을 레이드. 야생 몬스터와 같은 `mobs`/`attackRequest`/`partyKill` 채널 재사용.

**설계 요점**
- `MobSnapshot` 에 `type`/`villageId`/`guardMode` 추가 — 없으면 wildPredator(하위호환). `collectMobs` 가 `world.guards()` 도 수집, `spawnSyncedMob` 이 type 으로 분기해 게스트 측 경비를 `world.spawnGuard`(main 의 spawnKnight/Golem/RangedGuard 위임)로 생성. 경비도 `partyTransient`.
- **게스트 공격 가로채기**: 근접은 `attackKnight`(main.ts) 첫줄, 원거리/스킬은 combat.ts `isVillageGuard` 분기 진입 직후 `partyAttackIntercept`. 호스트가 `hostApplyGuestAttack` 으로 판정 — 피격 시 **마을 전체 enrage**(`enrageVillage`), 인라인 반격은 없음(반격은 호스트 guardAi 가 다음 틱에).
- **경비도 게스트를 노린다**: `updateVillageGuards` 가 `partyHostCombatTargets()` 로 같은 맵 가장 가까운 파티원(로컬+게스트)을 추격·공격, 원격이면 `partyDamageRemotePlayer`(패널 열림 보류).
- **처치 보상**: XP 는 같은 맵 전원 100%(야생과 동일), 전리품은 처치자에게 **철**(`partyKill.lootItem`). 호스트 직접 처치는 combat 경로로 철을 받고 파티엔 XP 만 공유.
- **게이트**: 게스트의 `updateVillageGuards`·경비 리스폰을 `partyWorldGuestActive` 로 정지(스냅샷 기반). 합류 시 게스트 로컬 경비는 sweep+stash, 탈퇴 시 복원.
- main.ts 배선은 전부 in-place — 라쳇 9489/459 순증 0.

**알려진 한계**
- villager/villageKing(평화 NPC)는 미동기화 — 게스트가 자기 로컬 주민을 쳐도 호스트 마을은 안 깬다. 레이드는 **경비를 직접 공격**하면 시작된다(enrage 동기화됨).
- 마을 위치는 비결정적(5차 지형 한계와 동일) — 게스트 화면의 호스트 경비는 게스트 자기 마을 건물과 어긋날 수 있다. 같이 싸우는 데는 무관(둘 다 호스트 곁에서 전투).
- 경비 공격 모션(휘두름)은 미전송 — 게스트는 경비의 이동·접근으로 위협을 인지.

## 13. 동굴·보물상자 공유 (7차)

> 정적 오브젝트(동굴 입구 `cave` + 보물상자 `chest`)도 호스트 권위로 공유 — 둘이 같은 상자를 보고 같이 연다. `mobs`/신규 `openRequest`·`chestLoot` 채널.

**설계 요점**
- `MobSnapshot` 에 `objType`/`opened` 추가. `collectMobs` 가 `world.chests()`·`world.caves()` 도 수집(위치 고정·hp 더미). `spawnSyncedMob` 의 정적 분기가 `world.spawnChest`/`world.spawnCave` 로 게스트 측 생성(`partyTransient`). 정적은 mobTargets 가 없어 보간·공격모션에서 자동 제외.
- **상자 개봉 = 호스트 권위**: 게스트가 동기화 상자를 열면 `partyGuestOpenIntercept` 가 `openRequest` 송신(로컬 개봉 생략). 호스트가 `markChestOpened`(opened+틴트+8초 만료) 후 **1회만** 전리품을 롤(`rollChestLoot`)해 **여는 사람에게만** `chestLoot` 전달 — 이중 지급·전리품 불일치 방지. 개봉 상태는 스냅샷으로 전파(양쪽 틴트), 8초 후 호스트 만료 → 스냅샷 diff 로 양쪽 제거.
- **게이트**: 합류 시 게스트 로컬 상자·동굴 sweep+stash(겹침 방지), 탈퇴 시 복원 — 상자는 `opened` 도 stash·복원해 **이미 연 상자가 미개봉으로 부활(재개봉=이중 전리품)** 하지 않는다. stash 상한 256(야생~48+경비~17+정적 누적분 여유). 동기화 정적은 `expirySweep` 에서 제외(호스트 스냅샷이 수명 관리). 세이브엔 `partyTransient` 로 영속 금지.
- **충돌·사운드 정합**(검수 반영): 동기화 정적은 위치 비결정이라 게스트 지형의 물·건물과 겹칠 수 있어 `collidable=false`(입구는 시각 참조·상호작용만, 이동 충돌 X). 개봉 사운드는 **여는 사람 로컬에서만** 재생 — `markChestOpened`(원격 전파·호스트 위임 처리)는 사운드를 울리지 않아 원거리 유령 효과음·이중 재생이 없다. 게스트 개봉 요청은 1.5초 쿨다운으로 호스트 무응답 시 재요청·재사운드 스팸을 억제.
- `applyMobsSnapshot` 정적 sweep 은 제너레이터 직접 순회(스프레드 제거)로 8Hz 핫패스 할당 0 유지.
- main.ts 배선은 전부 in-place — 라쳇 9489/459 순증 0(openChest 리팩터·사운드 한 줄 제거로 오히려 감소).

**알려진 한계**
- 동굴 **내부**는 비공유 — 게스트가 동굴 입구로 들어가면 자기 로컬 동굴(광물·내부 상자)을 탐험한다. 같은 입구를 보지만 안은 각자. 내부 co-op 은 별도 대형 작업(8차 후보).
- `mineChest`(동굴 내부 상자)는 overworld 공유 대상이 아니다(동굴 내부 전용).
- 호스트가 동굴/타이틀(inGame false)일 때 게스트 개봉 요청은 보류된다(다음 스냅샷이 상태 정정). 쿨다운으로 스팸은 막지만, 호스트 부재 중 실패를 알리는 부정 ACK 는 아직 없음(개봉자에게 명시적 실패 메시지 미표시).
