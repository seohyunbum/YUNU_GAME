# 잉여 고기·가죽 소모처(sink) — 설계 (구현 대기)

> 상태: **설계 / 구현 대기**. 코드 라인은 동시 작업으로 이동 가능 → 심볼명 기준.

## 0. 진단 (왜 쌓이는가)
- **고기**: 사냥하면 계속 나오는데, 소모는 ① 배고픔 채우기(1개=1칸, 최대 5라 금방 참) ② 크래프팅은 **고급 구급상자(고기 2)뿐**. → 사실상 무한 적체.
- **가죽**: 소모는 가방(7)·확장가방(14)·가죽갑옷(8)·침대(3)·활류(2~3)·고급구급상자(3) — **거의 전부 1회성**. 초반 지나면 적체.
- 요리/덫/펫먹이 시스템 **없음**(grep 0). 상점 판매는 가능(둘 다 SELL_SHOP_OFFERS 에 있음)이나 포인트만 됨.
- **핵심**: 1회성 제작템을 아무리 추가해도 "한 번 만들면 끝"이라 적체는 못 막는다. **반복 소모(consumable/변환)** 가 있어야 상시 빠진다.

## 1. 설계 원칙
1. **상시 sink는 "반복 소비되는 것"** — 회복식·버프식·덫·펫먹이처럼 계속 쓰는 것. 고렙 장비(1회성)는 "대량 흡수 1회"라 보조용.
2. **컨셉 정합**: 고기=요리(회복/버프), 가죽=가공·결속(무두질→장비·덫·붕대).
3. **기존 밸런스 보호**: 새 회복식이 medkit(회복15/refined_iron4)·고급구급(25)을 무력화하지 않게 — "싸고 양 많지만 효율은 낮게".
4. **구현 비용 최소 우선**: HEAL_ITEMS·레시피만 건드리는 MVP를 먼저, 버프/덫(신규 시스템·세이브·main.ts)은 후속.

---

## 2. 고기(meat) sink — "요리"

### 2-A. 구운 고기 (MVP, ★최우선) — 반복 회복식
- 레시피(확장 제작대 또는 신설 모닥불): **고기 6 + 석탄 1 → 구운 고기 1**, 회복 **10 HP**.
- 근거: medkit(15/refined_iron4)보다 회복 적지만 **흔한 고기+석탄**이라 대량 양산 → 잉여 고기 상시 소진. 사냥 루프와 자연스럽게 맞물림.
- 구현: `items.ts`(ITEM_NAMES `roast_meat`, ITEM_TIER common, sortCategory 소비) + `HEAL_ITEMS.roast_meat = 10` + `recipes.ts` 1개 + 손/핫바 비주얼(heldItemVisuals). **main.ts·세이브 변경 0**(HEAL_ITEMS·useHotbarItem 회복 경로 그대로 재사용).

### 2-B. 고기 스튜 (후속) — 전투 버프식 (가장 강한 상시 sink)
- 레시피(확장 제작대): **고기 16 + 물 양동이 1 + 제련된 다이아 1 → 고기 스튜 1**.
- 효과: 사용 시 회복 + **3분간 최대체력 +N·공격 +N 버프**. 보스전마다 만들어 먹게 → 고기 대량 상시 소비.
- 구현 비용(중): `skillBuffs` 패턴의 신규 버프 필드 + 타이머 + 스탯 곱/가산 지점(empowerMultiplier 와 동일 자리) + 소비 처리. 버프는 휘발(세이브 무영향).
- 결정 필요: 기존 empower(심판의빛)·목걸이 버프와 **스택 허용?**(권장: 곱연산 스택).

### 2-C. (선택) 소환사 펫 먹이 — 직업 전용 sink
- **고기 10 → 펫 경험치 대량**. summoner 만 해당이라 보조. companionProgress 에 XP 가산(이미 존재). 소량 plumbing.

---

## 3. 가죽(leather) sink — "무두질 + 사냥꾼 장비/소모품"

### 3-A. 무두질 가죽 (cured_leather) — 중간재(잉여 압축)
- 레시피: **가죽 8 + 석탄 1 → 무두질 가죽 1** (refined_iron 류의 상위 중간재).
- 단독으론 sink 아님(중간재) — 아래 3-B/3-C 가 이걸 대량 소비해야 의미. refined intermediate 라 `REFINED_*` 패턴 재사용.

### 3-B. 사냥꾼/드래곤 가죽 장비 (고렙, 1회성 대량 흡수)
- 예: **강화 가죽 갑옷**(무두질 가죽 12 → 방어 + 부가효과) / **드래곤 가죽 망토**(무두질 가죽 10 + 용의 비늘 2 → 이동속도/탐지). 컨셉상 가죽 다량.
- 1회성이라 "한 방에 많이 빼주는" 보조 sink. 구현: 장비면 ARMOR_VALUE/효과 지점 + 레시피.

### 3-C. 가죽 소모품 (반복 sink)
- **가죽 붕대(bandage)**: 가죽 4 → 회복 6(저효율·대량). HEAL_ITEMS 재사용 → MVP급 저비용 상시 가죽 sink. (구운 고기와 쌍으로 가면 고기·가죽 둘 다 상시 빠짐.)
- **사냥용 덫(trap, 후속)**: 무두질 가죽 2 + 막대기 3 + 철 1 → 설치형 덫(몬스터 속박/데미지). 소모성 → 상시 sink + 신규 gameplay. 구현 비용(큼): placeable(PLACEABLE_TYPES) + 월드오브젝트(세이브) + AI 피해 판정.

### 3-D. 명품 가방 3단계 (1회성)
- big_bag(64) 위 단계 — 잉여 가죽 대량 흡수하나 1회성. 우선순위 낮음.

## 4. 권장 우선순위 (비용 대비 효과)
| 순위 | 항목 | sink 유형 | 구현비용 | main/세이브 |
|---|---|---|---|---|
| ⭐1 | 구운 고기(2-A) | 반복(회복) | 낮음(레시피+HEAL_ITEMS) | 0 |
| ⭐2 | 가죽 붕대(3-C) | 반복(회복) | 낮음 | 0 |
| ⭐3 | 무두질 가죽(3-A)+강화 가죽 갑옷(3-B) | 변환+1회 대량 | 중간(레시피+장비효과) | 소 |
| 4 | 고기 스튜 버프식(2-B) | 반복(버프) | 중(버프 plumbing) | 휘발버프 |
| 5 | 사냥 덫(3-C) / 펫먹이(2-C) | 반복(신규) | 큼(placeable·AI) | 세이브 |

**MVP 권장 = 1+2** : `items.ts`+`recipes.ts`+`heldItemVisuals` 만으로 고기·가죽 **둘 다 상시 회복식 sink** 확보(main.ts·세이브 무변경, 레시피 사용 흐름 그대로). 그 다음 3(무두질+강화갑옷)로 "고렙 대량 흡수" 추가, 4·5는 더 큰 시스템.

## 5. 밸런스 메모
- 회복식 수치: 구운 고기 10 / 가죽 붕대 6 — medkit(15)·고급구급(25) 아래로 두어 상위 회복템 가치 유지. "싸고 흔하지만 효율 낮음" = 잉여 소진용.
- 양산 방지 불필요(목적이 소진). 단, 회복식이 너무 강하면 전투 난이도 붕괴 → 위 수치/쿨타임(기존 HEAL_ITEM_COOLDOWN_MS 1초) 유지.
- 고기 스튜 버프 수치는 보스 난이도 보고 보수적으로(최대체력+5/공격+3, 3분).

## 6. 구현 표면 요약 (MVP 기준)
- `items.ts`: `roast_meat`·`leather_bandage` → ITEM_NAMES, ITEM_TIER(common/uncommon), `HEAL_ITEMS`, itemSortCategory(소비=4).
- `recipes.ts`: 레시피 2개(확장 제작대 불필요 — 일반 제작대 가능하게 두면 접근성↑; 단 모닥불 신설 시 그쪽).
- `heldItemVisuals.ts`: 두 아이템 모델(없으면 돌덩이 폴백 + content-test 실패 → 전용 모델 필요, memory `yunu-game-item-visuals` 참조).
- `objectives.ts`(선택): "구운 고기 만들기" 가이드 퀘스트(요리 발견 유도) — countItem 기반, 소급 자동.
- 세이브/마이그레이션: **변경 없음**(순수 인벤토리 아이템).

## 7. 검증 계획 (구현 후)
- content-test: 새 아이템이 ITEM_NAMES·held 비주얼·레시피 재료 실존 통과.
- E2E: 레시피로 구운고기/붕대 제작 → 사용 시 체력 회복(HEAL_ITEMS 경로) → 고기/가죽 인벤토리 감소 확인.
- 밸런스: 회복량이 medkit 미만인지, 쿨타임 동작.
- verify+build 녹색.

## 8. 미결 결정 (구현 전 확인)
1. **모닥불(신설 설치물) vs 기존 제작대 재사용** — MVP는 제작대 재사용 권장(저비용). 분위기 원하면 모닥불 후속.
2. **고기 스튜 버프식 포함?** — 가장 강한 고기 sink지만 버프 plumbing 필요. v1 포함 vs v2.
3. **가죽 덫(신규 gameplay) 포함?** — 큰 작업. v2 권장.
4. **회복 수치/효율** — 위 제안(10/6) 확정 여부.
