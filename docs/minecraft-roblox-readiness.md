# Minecraft/Roblox 취향을 반영한 준비 점검

## 결론

현재 `C:\ai-game-lab`은 로컬에서 빠르게 실험하기 좋은 첫 환경입니다. 다만 아이가 Minecraft와 Roblox를 좋아한다면 게임 아이디어가 3D 공간, 블록, 아바타, 꾸미기, 역할 놀이, 장애물 코스로 흐를 가능성이 큽니다. 그래서 현재 프로젝트에는 2D 게임용 Phaser와 함께 3D 실험용 Three.js도 준비해두었습니다.

## 지금 준비된 것

- 로컬 브라우저 실행 환경: Vite
- 게임 로직: Phaser
- 3D/블록형 실험 준비: Three.js
- 메뉴, 설정, 결과 화면 확장 준비: React
- 타입 안정성: TypeScript
- 버전 저장: Git
- 아이디어 정리 문서: `docs/game-idea-canvas.md`
- AI 요청 예시: `docs/ai-prompts.md`

## 이 PC에서 확인된 관련 앱

- Minecraft Launcher: 설치됨
- Minecraft: Java Edition: 설치됨
- Minecraft for Windows: 설치됨
- Roblox Player: 설치됨
- Roblox Studio: 아직 설치되지 않음

## 추천 경로

### 1단계: 로컬 웹 미니게임

가장 추천하는 시작점입니다. Codex나 Claude Code가 파일을 직접 고치고, 브라우저에서 즉시 확인할 수 있습니다.

좋은 첫 주제:

- 블록을 밟으면 점수가 오르는 장애물 코스
- 자원을 모아 문을 여는 미니 어드벤처
- 색깔 블록을 정해진 순서로 밟는 퍼즐
- 아바타가 별, 보석, 재료를 모으는 수집 게임

### 2단계: 3D 블록 프로토타입

아이가 "진짜 Minecraft처럼 공간을 돌아다니고 싶다"고 말하면 Three.js로 작게 시작합니다.

첫 버전 범위:

- 평평한 바닥
- 큐브 10개 놓기
- 캐릭터 또는 카메라 이동

처음부터 채굴, 인벤토리, 멀티플레이, 무한 월드는 넣지 않습니다.

### 3단계: Roblox Studio

아이가 Roblox 안에서 직접 만들고 싶어 하면 Roblox Studio를 설치해서 별도 경로로 진행합니다.

주의할 점:

- Roblox Studio는 온라인 계정과 플랫폼 사용이 전제입니다.
- 공개/공유 전에 보호자 설정과 개인정보 노출 여부를 확인해야 합니다.
- 스크립트 언어는 JavaScript/TypeScript가 아니라 Luau입니다.
- Codex 프로젝트와 파일 구조가 달라서 별도 작업 흐름이 필요합니다.

## 부모-자녀 진행 체크리스트

- 오늘 만들 재미를 하나만 고른다.
- 게임 이름, 학교 이름, 친구 이름 같은 개인정보는 넣지 않는다.
- AI가 만든 코드는 부모가 실행 전에 한 번 확인한다.
- 아이가 말한 규칙을 먼저 적고, 구현은 작게 줄인다.
- 30분 안에 화면에서 움직이는 결과가 나오도록 한다.
- 재미있어진 순간마다 Git 커밋으로 저장한다.

## 지금 부족한 것

- Roblox Studio는 아직 설치되지 않았습니다.
- Minecraft 모드/애드온 제작 환경은 아직 구성하지 않았습니다.
- 소리 파일과 그림 파일 같은 게임 에셋 폴더는 아직 없습니다.
- 3D 샘플 화면은 아직 만들지 않았습니다.
