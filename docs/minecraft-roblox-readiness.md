# Minecraft/Roblox 취향을 반영한 준비 점검

## 결론

현재 `C:\ai-game-lab`은 로컬에서 빠르게 실험하기 좋은 첫 환경입니다. 아이가 Minecraft와 Roblox를 좋아한다는 점을 반영해, 첫 게임은 Three.js 기반의 3D 1인칭 야생 생존 게임으로 전환했습니다.

## 지금 준비된 것

- 로컬 브라우저 실행 환경: Vite
- 게임 로직과 렌더링: Three.js
- 메뉴, 설정, 결과 화면 확장 준비: React
- 타입 안정성: TypeScript
- 버전 저장: Git
- 시각 검증: Playwright Core + 로컬 Chrome/Edge
- 아이디어 정리 문서: `docs/game-idea-canvas.md`
- AI 요청 예시: `docs/ai-prompts.md`

## 이 PC에서 확인된 관련 앱

- Minecraft Launcher: 설치됨
- Minecraft: Java Edition: 설치됨
- Minecraft for Windows: 설치됨
- Roblox Player: 설치됨
- Roblox Studio: 아직 설치되지 않음

## 현재 게임에 반영된 취향

- Minecraft 느낌: 나무 채집, 제작대, 제련대, 동굴, 광물, 흑요석, 도구 제작
- Roblox 느낌: 1인칭 이동, 월드 탐험, 상호작용 오브젝트, 마을과 기사, 점진적 장비 성장
- 부모-자녀 작업 흐름: 로컬 브라우저에서 바로 실행하고 Codex가 파일을 직접 수정

### Roblox Studio

아이가 Roblox 안에서 직접 만들고 싶어 하면 Roblox Studio를 설치해서 별도 경로로 진행합니다.

주의할 점:

- Roblox Studio는 온라인 계정과 플랫폼 사용이 전제입니다.
- 공개/공유 전에 보호자 설정과 개인정보 노출 여부를 확인해야 합니다.
- 스크립트 언어는 JavaScript/TypeScript가 아니라 Luau입니다.
- Codex 프로젝트와 파일 구조가 달라서 별도 작업 흐름이 필요합니다.

## 부모-자녀 진행 체크리스트

- 오늘 만들 재미를 하나만 고른다.
- 게임 이름, 학교 이름, 친구 이름 같은 개인정보는 넣지 않는다.
- AI가 만든 코드는 부모가 실행 전에 한 번 확인한다.
- 아이가 말한 규칙을 먼저 적고, 구현은 작게 줄인다.
- 30분 안에 화면에서 움직이는 결과가 나오도록 한다.
- 재미있어진 순간마다 Git 커밋으로 저장한다.
