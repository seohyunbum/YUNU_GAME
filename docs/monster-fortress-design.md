## 0. 핵심 아키텍처 판단 (코드 근거)

- **별도 맵 X / 동굴형 던전 O** — 신규 요새는 들어가 웨이브를 막고 죽거나 '나가기'로만 나오는 **격리 인스턴스**. cave 라이프사이클(진입/이탈·복귀위치·격리 오브젝트셋·리스폰 억제)이 정확히 부합. 오버월드 맵(WORLD_MAPS)은 탐험형이라 부적합·고부담.
- **`locationMode` 확장 X / "cave" 재사용 + `fortressSiege` 상태 플래그 O** — `locationMode === "cave"` 분기가 40여 곳이라 새 enum 추가는 전수 감사+세이브 마이그레이션 부담. 신규 요새도 `locationMode="cave"` 로 두고, 별도 `fortressSiege` 상태로 웨이브·드랍억제·이탈차단을 분기 → **세이브 마이그레이션 0**.
- **리네임은 사용자 텍스트만** — `fortressBossKills` 는 `tutorialSignals`(휘발, 세이브 미저장)라 마이그레이션 불필요. 내부 심볼(`fortressMonster/fortressBoss/fortressLevel/buildFortressDecor/spawnFortressMonster`)은 유지하고 신규 요새는 별도 명명(`siege*`)으로 충돌 회피.

## B. 신규 몬스터 요새 (웨이브 디펜스 아레나)

### B1. 성격
- **플레이어 자가 방어 호드 생존**: 중앙 아레나에 플레이어, 4갈래 통로 입구에서 몬스터 웨이브가 시간차로 몰려와 플레이어에게 수렴. (※ 설치형 방어물 건설은 비포함 — 결정 Q4)
- 한 웨이브 전멸 = 클리어 → 잠시 후 다음 웨이브. 단계(stage)의 모든 웨이브 클리어 = 단계 클리어 → **보상 지급** → 다음(더 어려운) 단계.

### B2. 아키텍처 (데이터/리프 우선)
```
game/fortressSiege.ts (신규 리프)
  · ARENA_STAGES 데이터 테이블(단계별 웨이브 수·마릿수·레벨오프셋·보상)
  · 웨이브 상태머신: SiegeState { stage, waveIndex, spawnQueue[], aliveIds, phase }
  · updateSiege(ctx, delta): 스폰 타이머·웨이브 클리어 판정·단계 진행 (순수 로직, 스폰/제거는 ctx 콜백)
  · 보상 계산(전직의서 수량·단계 아이템)
game/interiors.ts
  · createSiegeArenaInterior(ctx): 정사각 아레나 셸(중앙 단상+4 방사 통로+성벽·화로·깃발) — 기존 fortress 데코 자산 재사용 + 인스턴싱
game/caveMonsters.ts
  · spawnFortressMonster 재사용(또는 siege 인자 추가). AI 클램프 경계를 아레나 경계로 파라미터화
main.ts (배선만)
  · fortressSiege 상태 필드 + enterFortressSiege()/exitFortressSiege() 얇은 위임
  · update 루프에서 siege 활성 시 updateSiege() 호출(기존 cave 경로 옆)
  · 드랍 억제·이탈 차단·저장 차단 게이트(각 1줄)

  · 단계/웨이브/남은 몬스터 뷰모델 + 렌더
```

### B3. 아레나 레이아웃 (≈40×40 정사각, cave 인스턴스 내)
- 동굴 복도(16×190)는 좁아 방사형 부적합 → **전용 정사각 아레나 셸** 신규(중앙 단상 platform + N/S/E/W 4 통로 입구 + 성벽 ramparts). 좌표는 cave 인스턴스 공간에 배치, 몬스터 AI 클램프는 아레나 경계(±ARENA_HALF) 사용.
- 플레이어 진입 = 중앙 단상. 몬스터 = 4 통로 입구에서 스폰 → 중앙으로 추격.

### B4. 웨이브·단계 (무한 점증 — 결정 Q2 = 무한 단계)
끝이 없는 점증형. 단계 N(1,2,3…)이 오를수록 강해지고, 죽거나 '나가기'로만 종료. 공식(조정 가능):
- 웨이브 수/단계: `min(2 + floor(N/2), 5)`
- 단계 기준 레벨: `base + N*3` (base = 요새 입구 region 레벨 또는 플레이어 레벨)
- 웨이브당 마릿수: `~6 + N` (단, **동시 생존 상한 ~12** — perf, 초과분은 다음 웨이브로 이연)
- 종 구성: N↑ → 고렙 변종 비중↑, 주기적으로 정예/미니보스
- **전직의서/단계**: `1 + floor((N-1)/3)` → 1~3단계 1개, 4~6단계 2개, 7~9단계 3개 …
- **기타 보상/단계**: N에 따라 흑요석→다이아→제련된 다이아→날카로운 흑요석으로 점증
- 보상은 **단계 클리어 즉시 지급** → 다음 단계에서 죽어도 이전 보상 보존(자연스러운 푸시-유어-럭).

### B5. 보상
- 단계 클리어 = **전직의서 무조건 지급(단계별 수량 차등)** + 단계별 아이템. `addItem` + `showMessage` 재사용(기존 보스 드랍 패턴).
- 데이터: `ARENA_STAGES[stage].rewards = { tomes, items: {...} }`.

### B6. 사망·이탈·저장
- **요새 내 사망 = 아이템 0 드랍**: `dropInventoryOnDeath` 호출을 siege 활성 시 스킵(드랍 억제). 사망 = 요새 이탈(오버월드 복귀, 손실 없음). 이미 받은 단계 보상은 유지.
- **이탈 차단**: siege 진행 중 일반 이탈(E) 차단. 이탈 경로 = ① 사망 ② '나가기/포기' 명시 버튼(단계 사이 또는 언제든) ③ 모든 단계 클리어(승리).
- **저장 차단**: siege 중 저장 불가(메시지 안내). siege 상태는 휘발 → 저장/재로드 시 일반 동굴로 폴백(마이그레이션 0). (대안: siege 진행 저장 — 복잡도↑, 비채택)

### B7. 진입 (결정 Q1)
- 권장: 오버월드에 등장하는 **전용 '몬스터 요새 입구' 구조물**(동굴 입구와 유사한 상호작용으로 진입). 발견형·재방문 가능. 위치=희귀 스폰 또는 고정 지점.

### B8. HUD
- 상단/중앙에 "단계 N/5 · 웨이브 M · 남은 몬스터 K" + 단계 클리어/다음 웨이브 카운트다운. hudRenderer 뷰모델 확장.

### B9. 비주얼 (요새 한가운데 느낌)
- 중앙 단상(돌+룬) + 4 통로 아치 입구 + 성벽(ramparts) + 화로·깃발·해골더미·바닥 가시(기존 fortress 데코 자산 재사용) + 붉은 조명/앰비언트.
- **perf**: 반복 props(화로·가시·횃불)는 InstancedMesh(biomeDecor 패턴)로 draw call 억제. 공유 geometry/material 모듈 1회 생성. PointLight 수 상한(거리 컬링). 스폰은 프레임당 2~4마리 스태거(히치 방지). `perf-check` 전후 비교.

## C2. 적대적 검증(2026-06-19) — 확정 수정 6건

배포 후 6차원 적대적 코드 리뷰 + 발견별 반증 검증(16건 발견→10 확정). 수정:
- **엘리트 공격력 누락**: HP·크기 ×1.4 인데 공격력 미적용 → 공격력도 ×1.4.
- **상태 일관성(맵 텔레포트·파티 소환)**: siege 중 `teleportToWorldMap`/지도 패널 차단 + `leaveCave` 가 어떤 경로로 나가든 `fortressSiege=null` 해제(소환·텔레포트가 exitFortressSiege 우회하던 근본 버그).
- **독수리 빙의 정리**: siege 이탈 시 빙의 미해제·eagle 미정리(orphan) → `leaveCave` 가 `endEaglePossession` 호출 + `spawnEagleSummon` 이 동굴 소환 eagle 을 caveObjectIds 추적.
- **spawnCursor 누적**: 모듈 전역이라 재진입마다 통로 분배 편향 → `SiegeState` 로 이동(진입마다 리셋).
- 의도 유지(미수정): **요새 몬스터 일반 루팅은 보너스로 허용**(사용자 요구는 "플레이어 사망 무드랍"이며 몬스터 루팅 아님). 단계 클리어 보상은 별도. fortressGate 영속은 정상(재진입 시 1단계부터=의도).
