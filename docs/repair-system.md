# 수리 시스템 설계 (Repair System)

> 상태: **P1 구현 완료** (2026-06-11). P2(방패 등 확장)는 미착수.
> 관련 코드: `src/game/items.ts` `repairMaterialFor`/`repairPerMaterial`/`toolMaxDurability` · `src/ui/workbenchPanel.ts` 수리 섹션 · `src/main.ts` `repairToolSlot`/`wornToolSlots`.
> 구현 시 변경점: 회복량은 floor 가 아니라 **ceil(최대 × 0.5)** — 홀수 내구도(철 45, 금 25)에서도 "재료 2개로 완전 회복" 약속을 지키기 위함. `test:content` 가 이 불변식을 강제한다.

## 1. 목적
좋은 도구가 내구도 소진으로 **사라지는 것을 줄인다.** 제작대에서 재료를 들여 내구도를 회복해, 비싼 도구를 오래 쓰게 한다.

## 2. 확정 결정
- **수리처 = 제작대**(제작대 패널 안 "수리" 섹션). 신규 설치물·NPC 미사용.
- **파손 정책 = 기존 유지**: 내구도 0 도달 시 **아이템 파손·삭제**(현 동작 그대로). "부서진 상태로 보존" 옵션은 **미채택**(근거 §8).
- 따라서 수리는 **0에 도달하기 *전에*** 하는 예방 행위다. 0이 되면 그대로 잃는다.

## 3. 현재 내구도 모델 (배경 · 자기완결)
- **내구도 대상**: 도끼·곡괭이·삽 (`DURABLE_TOOL_TABLES = [AXE_POWER, PICKAXE_POWER, SHOVEL_POWER]`).
  **전투 무기**(검·단검·활·마법봉·권총·소총·지팡이)는 **내구도 없음(무한)**.
- **소비**: 채집/굴착 사용 1회 = 내구도 1 (`consumeDurability`). 전투 타격은 소비 안 함.
- **파손**: 사용 횟수가 최대치 도달 → 슬롯에서 아이템 삭제 + "○○의 내구도가 다해 부서졌습니다".
  저내구 경고는 남은 횟수 ≤ `min(5, 최대의 20%)` 구간에서 출력.

| 등급 | 도끼·곡괭이·삽 최대 내구도(=사용 횟수) | 비고 |
|---|---:|---|
| 나무(약한/날카로운) | 10 | |
| 돌 | 20 | |
| 구리 | 30 | |
| 금 | 25 | ⚠️ 구리보다 낮음 — 의도된 "약체" 티어 |
| 철 | 45 | |
| 다이아 | 80 | |
| (미등록) | 10 | `DEFAULT_TOOL_DURABILITY` |

## 4. 수리 메커니즘
- **위치**: 제작대 패널 "🔧 수리" 섹션. 인벤토리의 *닳은* 도구를 나열 — `내구도 N/최대` + 비용 + [수리] 버튼.
- **비용**: 도구 **등급의 제련 재료 1개 / 수리 1회**(예: 철 도구 → `refined_iron` 1).
- **효과**: 수리 1회 = **최대 내구도의 50% 회복**(상한 = 최대). 완전히 닳은 도구는 **2회**로 완전 회복.
- **비활성 조건**: 재료 부족 또는 내구도가 이미 최대.

## 5. 데이터 / 함수 (leaf · 데이터-주도)
도구 id가 `{prefix}_axe` 꼴이라, 등급 재료·회복량을 **함수로 파생**(per-item 테이블 불필요):

```ts
// src/game/items.ts (구현됨)
// 도구 prefix(wood/stone/copper/iron/gold/diamond) → 수리 재료(refined_X). MATERIALS 재사용.
export function repairMaterialFor(item: ItemId): ItemId | null { /* prefix → refined, 없으면 null */ }
// 수리 1회 회복량 = ceil(최대 내구도 * 0.5) — 홀수 내구도도 2회로 완전 회복
export function repairPerMaterial(item: ItemId): number {
  return Math.ceil(toolMaxDurability(item) * 0.5);
}
```

→ 향후 **방패(탱커)·신규 내구도 아이템**도 이 함수에 분기 한 줄만 추가하면 자동 수리 대상(예: `iron_shield` → `refined_iron`).

## 6. 후크 위치 (구현 시)
| 조각 | 위치 | 내용 |
|---|---|---|
| 파손 로직 | `consumeDurability`(L7279) | **변경 없음** — 파손·삭제 정책 유지 |
| 수리 실행 | 신규 `repairItem(slot)` (main.ts) | 재료 차감 → `slot.durabilityUsed` 감소(상한 0) + 연출/소리 |
| 수리 UI | `src/ui/workbenchPanel`(이미 분리, leaf) | "수리" 섹션 렌더 + 버튼 |

→ main.ts 변경 최소(`repairItem` + 배선)이고 나머지는 leaf. §12(멀티에이전트) 친화적.

## 7. 밸런스
- **완전 수리(재료 2개)** < **재제작**(보통 제련재료 3개 이상 + 막대) → 수리 이득 = *재료 절약 + 도구 유지*.
- 비싼 도구일수록 이득 큼(다이아 80내구) → **"좋은 도구 보존"** 목적에 정합.
- 회복 계수 **0.5**로 난이도 조절(↑ 쉽게 / ↓ 빡세게). **무료 수리는 미채택**(내구도 자체가 무의미해짐).
- 금 도구(내구도 25 < 구리 30)는 수리 수요가 높음 — 의도된 약체 티어 특성.

## 8. 미채택 대안 (결정 기록)
- **"0에서 삭제 대신 부서진 상태로 보존(수리 가능)"**: 좋은 도구 *영구* 보존엔 강력하나, **기존 파손의 긴장/손맛을 유지**하기 위해 채택하지 않음. 내구도 관리는 플레이어 책임으로 둔다. (재검토 시 본 절을 갱신)

## 9. 거버넌스 / 저장
- **저장 변경 없음**: `slot.durabilityUsed` 는 이미 직렬화·복원됨 → 마이그레이션 불필요, `SAVE_VERSION` 불변.
- `test:content` 추가 권장: 모든 DURABLE 도구가 `repairMaterialFor` 로 **실존 재료**를 반환하는지(누락 시 실패).
- 데이터-주도라 main.ts 거의 불변 → 줄/메서드 래칫 영향 미미.

## 10. 단계
- **P1 ✅ 구현 완료**: 제작대 수리(도구) — `repairToolSlot`(main) + `repairMaterialFor`/`repairPerMaterial`(items) + 제작대 "도구 수리" 섹션(workbenchPanel). 닳은 도구가 있을 때만 섹션 노출.
- **P2**: 방패(탱커)·향후 내구도 아이템으로 확장(동일 함수 재사용).
